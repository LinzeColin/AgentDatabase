## 数据与安全模型

| 参数 | 当前值 | 原理 |
|---|---|---|
| raw storage | append-only | 已存在 raw 不覆盖、不修改、不删除 |
| raw_mutation | false | 所有 proposal/apply 和清理均排除 raw |
| credentials in Git | false | cookie、session、password、API key、private key 禁止提交 |
| public snapshot | redacted derived only | UI 不读取 private/raw 正文 |
| active write | proposal + human approval | 未批准不得进入 applying |
| rollback | required before apply | validation 失败必须恢复 transaction snapshot |
| secret error output | redacted | 用户错误不泄露敏感值 |
| source package | hash + restore contract | 原始包与派生实现分开管理 |

Proposal 当前状态机：

```text
draft -> pending_human_review -> approved_by_human -> applying
      -> applied -> validated -> committed
applying -> failed_validation -> rollback_or_needs_revision
```

允许 target types：memory、agents_rule、config、formula、ui_text、taxonomy、
report_template。禁止 target types：raw_archive、public_raw、credentials。

核心公式：

```text

  (automation*2.0 + template*1.2 + productization*1.5 + loops*0.75) * 4.0)

automation_ratio = automation / max(1, automation + productization + template + compounding)
```

## 当前维护规则

- 参数变化必须先修改权威配置或 registry，再更新本摘要。
- 所有权重须记录单位、范围、公式、证据和 rollback；未校准值必须标记 heuristic。
- UI what-if 只生成预览，不直接改 active config。
- raw、credential、deployment authorization 和 GitHub delivery 不是可调评分参数。
- v1.2.1 完成前不得把局部 PASS、历史 R8 或字符串 validator 解释为整体完成。
- 本次 `S03-P1-T2` 只收敛文档，不 push、不 deploy、不删除文件、不清缓存。
