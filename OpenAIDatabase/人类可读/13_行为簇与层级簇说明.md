# 行为簇与层级簇说明

## 当前结论

S06 P1 已完成。任务 ID 为 `MA-V12-S06P1`，验收 ID 为 `ACC-MA-V12-S06P1`，
状态为 `phase_s06_p1_cluster_builder_completed_pending_s06_p2`。

`scripts/build_memory_atlas_clusters.py` 从
`data/derived/behavior_intelligence/events.json` 读取 S05 canonical events，生成
`data/derived/behavior_intelligence/clusters.json`。该文件服务于后续低价值循环、
机会发现、ROI 和可视化，但本阶段只实现 Cluster builder。

## Cluster 类型

- 主题簇：围绕 project、topic、task 或关键词主题聚合事件。
- 层级簇：按 source、project、task、language 将主题簇组织成可探索层级。

每个 cluster 都包含中文摘要、事件数量、代表事件、时间范围、过滤维度和
`evidence_refs`。中文摘要只描述事件聚合，不输出个人状态判断，也不做无证重大结论。

## 过滤维度

S06 P1 支持 `source/time/project/task/language` 过滤合同。CLI 可通过
`scripts/atlasctl.py analyze --stage clusters --dry-run` 查看当前索引，也可用
`--source`、`--time-from`、`--time-to`、`--project`、`--task`、`--language`
生成过滤后的临时结果。

## 证据边界

Cluster 的 evidence refs 只来自已有 event 的 raw、manifest、derived 或 missing reason
引用。没有 evidence refs 的 cluster 不通过 validator。

下一步只允许进入 S06 P2。S06 P2 才会识别低价值循环和 Decision Debt Ledger。
