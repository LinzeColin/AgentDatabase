# 01 v1.2 四线14Stage升级总览

## 结论

v1.2 是一次四线、14 Stage 的整合升级。它不是新建一个分裂系统，而是在现有
OpenAIDatabase / Memory Atlas 上建立更清晰的人类入口、机器治理入口、数据同步、
行为智能、可视化、授权 apply 和运行维护能力。

S01 整体复审已通过。S02 P1 已完成数据源模型定义，下一步是 S02 P2 source registry。

## 四线

| 线 | 内容 | 目标 |
|---:|---|---|
| 1 | Anthropic 化行为智能 | 从 transcript 和 derived data 中形成 facets、clusters、latent signals、自我迭代和协作质量解释。 |
| 2 | 信息 ROI、全中文和多维可视化 | 打开系统先看到结论、变化、证据和下一步，而不是 ID、schema 和路径堆叠。 |
| 3 | ChatGPT、Codex、后续其他 agent 自动备份进 GitHub | 通过 source registry、raw append-only、manifest/hash 和 derived reports 建立公开备份链路。 |
| 4 | UIUX 多模态交互、参数公式治理、授权后自动 apply、运行维护简化 | 用 command palette、proposal gate、formula what-if 和 atlasctl 降低维护成本。 |

## 14 Stage 执行规则

- 每次 run 最多只完成一个 phase。
- 一个 Stage 的全部 phase 完成后，必须做 Stage 复审。
- 复审暴露的问题必须先解决，才能进入下一个 Stage。
- 整体完成后才上传 GitHub main。
- 整体完成后才重装 app 入口。
- 本机清理和缓存减压属于整体完成后的收尾工作。
