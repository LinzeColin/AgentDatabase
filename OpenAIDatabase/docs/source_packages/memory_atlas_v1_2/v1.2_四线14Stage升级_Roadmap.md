# 02 Stage / Phase / Task Roadmap（含 Pursuing Goal）

## 总规则

- Stage 总数：14。
- 每个 Stage 不超过 5 个 Phase。
- 每个 Phase 不超过 5 个 Task。
- Codex 每次只能执行一个 Stage。
- 每个 Stage 完成后必须停止并等待人类确认。
- 每个 Stage 都有 acceptance criteria、stop condition、pass gate、validation、rollback。

## S01｜需求冻结与双平面落地

**所属主线：** 基础/全局

**Pursuing Goal：** 锁定 v1.2 四线14Stage升级的需求、边界、文件结构和执行协议，避免 Codex 后续遗忘、越界或重复讨论。

### P1｜现状核验
1. 读取 OpenAIDatabase README、AGENTS、三件套和现有 Memory Atlas 入口
2. 确认当前 docs/governance、apps、scripts、tests 的实际位置
3. 列出现有硬边界中需要被 v1.2 替换的规则

### P2｜双平面创建
1. 新增或补齐 人类可读/ 目录
2. 新增或补齐 机器治理/ 目录
3. 保留根目录三件套作为 owner 门面
4. 不要移动现有运行代码

### P3｜需求冻结
1. 写入 v1.2需求冻结清单
2. 写入四线范围和14Stage只执行当前Stage规则
3. 写入 raw公开授权与凭证排除规则
4. 写入后续其他agent数据源扩展规则

### Acceptance Criteria
- 仓库内存在全中文 人类可读/00_快速入口.md
- 机器治理/ 中存在 v1.2 需求冻结配置
- 根目录三件套未被删除或降级
- README/人类入口能说明 v1.2 边界变更

### Stop Conditions
- Codex 试图移动 apps/scripts/tests/config 以迎合双平面
- Codex 删除现有 governance 或三件套
- Codex 将本 taskpack 大段写入 AGENTS.md

### Validation
```bash
python -B -m unittest discover tests -q || true
```
```bash
git status --short
```
```bash
人工检查 人类可读/ 是否全中文且不是跳转页
```

### Pass Gate
双平面存在、需求冻结清单存在、旧隐私边界已被明确替换为用户授权后的 raw 公开/只读/凭证排除边界。

### Rollback
revert 本 stage 新增的人类可读/、机器治理/ 文件；不得改动 raw。

## S02｜统一数据源注册：ChatGPT / Codex / 后续其他 agent

**所属主线：** 线3

**Pursuing Goal：** 建立通用 source registry，使 ChatGPT、Codex 与未来其他 agent 数据都能备份进 GitHub，而不是只做两个硬编码来源。

### P1｜数据源模型
1. 定义 source_id、source_type、agent_name、raw_root、sync_mode、public_backup_mode、connector_capability
2. 支持 chatgpt、codex、other_agent 三类
3. 规定每个 source 的 transcript 与 credential 区分

### P2｜注册表
1. 在 机器治理/同步与备份/ 建立 source registry
2. 加入 ChatGPT browser connector、official export fallback、Codex local sync
3. 加入 future_agent_adapter 模板

### P3｜人类说明
1. 在 人类可读/05_ChatGPT与Codex及其他Agent自动同步说明.md 写清楚同步范围
2. 说明后续 agent 如何接入
3. 说明哪些数据会进入 GitHub

### Acceptance Criteria
- source registry 不是仅 chatgpt/codex 硬编码
- 人类说明明确写出 ChatGPT、Codex、后续其他 agent 数据备份进 GitHub
- 每个 source 都有 transcript/credential 边界

### Stop Conditions
- 只实现 ChatGPT/Codex 并没有 future agent 接口
- 把 cookie/token 当作 agent 数据源内容
- 源注册表缺少 public backup mode

### Validation
```bash
python -B -m json.tool 机器治理/同步与备份/sync_source_registry.json
```
```bash
人工检查 source registry 是否含 future_agent_template
```

### Pass Gate
统一 source registry 可表示 ChatGPT、Codex、未来其他 agent，并明确公开备份策略。

### Rollback
删除或恢复 source registry 与相关说明文件，不影响 raw archive。

## S03｜Raw 明文公开归档与只读完整性

**所属主线：** 线3

**Pursuing Goal：** 实现 raw/transcript 明文进入公开 GitHub 的归档规则，同时确保 raw 只读、只追加、不覆盖、不增删改，并排除凭证。

### P1｜公开 raw 路径
1. 定义 public raw archive 路径
2. 定义 raw manifest/hash 文件
3. 定义 append-only 规则
4. 定义 hash drift fail 规则

### P2｜凭证排除
1. 实现 credential is not memory 的轻量检查
2. 只拦截 credentials，不拦截普通 transcript
3. credential 检查集成到 sync/audit，不新增复杂 UI

### P3｜机器账本
1. 生成 raw manifest 作为机器文件
2. 不要把 raw manifest 做成人类主要页面
3. manifest 应能映射 source/file/hash/imported_at

### Acceptance Criteria
- raw transcript 可明文进入公开路径
- raw manifest/hash 可生成
- 修改已有 raw 文件导致 validation fail
- 凭证 pattern 导致提交 gate fail
- 人类文件不被 raw manifest 细节污染

### Stop Conditions
- Codex 修改已有 raw 文件
- Codex 自动删除/覆盖 raw
- credential 检查被写成复杂新治理系统
- cookie/token 被提交

### Validation
```bash
python scripts/atlasctl.py audit --check raw-append-only || true
```
```bash
python scripts/atlasctl.py audit --check credentials || true
```
```bash
git diff -- raw_archive_path
```

### Pass Gate
raw 可公开备份，同时 append-only 和 credential exclusion 均可验证。

### Rollback
回滚脚本/config；新增 raw 文件不自动删除，由人类决定。

## S04｜自动同步 MVP：ChatGPT / Codex / 后续 agent

**所属主线：** 线3

**Pursuing Goal：** 建立一键同步 MVP：ChatGPT 已登录浏览器只读同步、official export fallback、Codex local sync、后续 agent adapter，并生成可提交 GitHub 的 raw/derived/reports。

### P1｜ChatGPT 同步
1. 实现或规划 browser connector 在用户已登录状态下只读读取
2. 遇到密码/验证码立即停止
3. 不得发送消息/删除/归档/重命名会话
4. 保留 official export ZIP/conversations.json fallback

### P2｜Codex 与其他 agent 同步
1. 复用或扩展现有 sync_codex_memory_data.py
2. 支持后续其他 agent adapter 的最小接口
3. 每个来源输出 raw + derived + run log

### P3｜GitHub 备份
1. 将 raw/derived/reports/run logs 纳入 git backup 流程
2. 支持手动 push 与定时/本地自动化
3. 失败时输出中文原因和 fallback 建议

### Acceptance Criteria
- 可运行 sync chatgpt/codex/future-agent 占位接口
- official export fallback 可用
- derived visualization 数据能更新
- GitHub 备份命令有 dry-run 和 apply 模式

### Stop Conditions
- 自动登录流程要求 Codex 获取密码/验证码
- connector 尝试修改 ChatGPT 会话
- sync 失败后继续生成伪数据
- 后续 agent 扩展被遗漏

### Validation
```bash
python scripts/atlasctl.py sync --source codex --dry-run || true
```
```bash
python scripts/atlasctl.py sync --source chatgpt --dry-run || true
```
```bash
python scripts/atlasctl.py build-atlas || true
```
```bash
git status --short
```

### Pass Gate
三类来源同步路径存在，且可 dry-run、fallback、构建 Atlas derived 数据。

### Rollback
回滚 connector/config；不删除已备份 raw，除非人类明确要求。

## S05｜Anthropic 化 Facet 抽取与事件语义层

**所属主线：** 线1

**Pursuing Goal：** 把 raw/derived 数据抽象成可聚类、可可视化、可证据追溯的行为 facets 与 canonical events。

### P1｜Facet schema
1. 定义 source、topic、intent、task_type、project、output_type、language、tool、turn_count、friction、value_signal
2. 加入 future_agent_source 字段
3. 字段英文，中文解释必须在人类文件中说明

### P2｜Extractor
1. 实现或扩展 facet extractor
2. 从 ChatGPT/Codex/other agent raw 或 derived 中抽取事件
3. 缺失字段要 graceful fallback

### P3｜证据引用
1. 每个 event 保留 source_id、raw_ref 或 manifest_ref
2. 不要求复杂 Raw-to-Insight Replay UI
3. 只实现轻量 evidence ref，满足可追溯目标

### Acceptance Criteria
- canonical event 可覆盖 ChatGPT/Codex/future agent
- 每条 event 有 evidence ref 或缺失原因
- 人类文件能解释 facet 含义
- 不输出纯机器字段给首屏

### Stop Conditions
- extractor 为缺失数据生成假记录
- 人类 UI 直接展示 schema 字段堆
- evidence ref 完全缺失

### Validation
```bash
python scripts/atlasctl.py analyze --stage facets || true
```
```bash
python -B -m json.tool data/derived/behavior_intelligence/events.json || true
```

### Pass Gate
行为事件与 facets 可被后续 cluster、ROI、latent、visualization 复用。

### Rollback
回滚 extractor 和 derived 输出；不修改 raw。

## S06｜行为簇、低价值循环与机会发现

**所属主线：** 线1

**Pursuing Goal：** 建立 Clio-like 主题簇、层级簇、低价值循环簇、机会簇，并让 Atlas 支持多角度探索。

### P1｜Cluster builder
1. 实现主题簇和层级簇
2. 支持按 source/time/project/task/language 过滤
3. 每个 cluster 有中文摘要和 evidence refs

### P2｜低价值循环
1. 识别重复返工、反复讨论未落地、过度优化、scope creep
2. 加入 Decision Debt Ledger
3. 加入 Action Half-Life

### P3｜机会发现
1. 识别可自动化、可产品化、可模板化、可复利、应该暂缓的机会
2. 为机会增加 为什么不是现在 卡片
3. 不接外部经济数据库

### Acceptance Criteria
- 至少有行为簇/低价值循环/机会三类输出
- 中文摘要可读
- 每条机会有证据、下一步、半衰期或暂缓理由
- cluster 不只是关键词列表

### Stop Conditions
- 没有证据就生成重大结论
- 输出心理诊断
- 把机会列表做成无穷压力清单

### Validation
```bash

```
```bash
python scripts/atlasctl.py audit --check insight-evidence || true
```

### Pass Gate

## S09｜潜性报告与自我迭代

**所属主线：** 线1/线4

**Pursuing Goal：** 生成证据化潜性报告、自我迭代建议、proposal expiry 和 decision debt，但不做心理诊断、不污染人类入口。

### P1｜Latent signals
1. 每条潜性信号包含主张、支持证据、反对证据、替代解释、置信度、下一步验证
2. 禁止人格/心理诊断
3. 使用 Evidence Strength Badge

### P2｜自我迭代
1. 生成 memory/config/AGENTS/style/personalization 建议
2. 加入 proposal expiry
3. 建议必须有 action half-life

### P3｜决策债
1. 建立 Decision Debt Ledger
2. 识别反复讨论未落地的问题
3. 给出最小下一步，不生成压力列表

### Acceptance Criteria
- 潜性报告有证据、反证、替代解释、置信度
- proposal 有有效期
- decision debt 有最小下一步
- 人类文件不过度堆叠分析术语

### Stop Conditions
- 输出心理诊断或人格标签
- 无证据高置信结论
- proposal 永久 pending 且无过期机制

### Validation
```bash
python scripts/atlasctl.py analyze --stage latent || true
```
```bash
python scripts/atlasctl.py audit --check latent-safety || true
```

### Pass Gate
潜性和迭代建议是证据化、可反驳、可行动、不过度心理化的。

### Rollback
回滚 latent/self_iteration derived 输出与配置。

## S13｜Proposal 授权 Apply、Diff Narrator 与回滚

**所属主线：** 线4

**Pursuing Goal：** 实现 proposal-only → 人类授权 → 自动 apply → validation → commit/rollback 的闭环，并用中文 diff narrator 降低审核成本。

### P1｜状态机
1. draft → pending_human_review → approved_by_human → applying → applied → validated → committed
2. 失败路径 failed_validation → rollback_or_needs_revision
3. proposal expiry 集成

### P2｜Diff narrator
1. 人类可读解释改了什么、为什么改、影响什么、如何验证、如何回滚
2. 不要生成冗长机器 diff 到人类首页
3. 机器 diff 保留在治理/证据文件

### P3｜Apply 与回滚
1. 人类授权后才能 apply
2. apply 后自动跑 validation
3. 失败后生成 rollback 或执行 rollback
4. raw archive 永不作为 apply target

### Acceptance Criteria
- 未授权 proposal 不会 apply
- 授权后可以自动 apply
- diff narrator 中文可读
- 失败可回滚
- raw 不被修改

### Stop Conditions
- 未授权自动 apply
- proposal 修改 raw
- 没有 rollback point
- 测试失败继续 commit

### Validation
```bash
python scripts/atlasctl.py proposals --dry-run || true
```
```bash
python scripts/atlasctl.py apply --proposal sample --dry-run || true
```
```bash
git diff --check
```

### Pass Gate
授权 apply 闭环可用，审核成本下降，回滚路径清晰。

### Rollback
使用 proposal rollback 或 git revert；raw 文件不回滚。
