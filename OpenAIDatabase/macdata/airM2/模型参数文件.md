# airM2 macdata 模型参数文件

## Codex Automation 参数

| 参数 | 当前值 | 说明 |
|---|---|---|
| codex_model | gpt-5.3-codex-spark | 自动化建议模型 |
| reasoning_effort | xhigh | 需要可靠预检、上传、清理门控 |
| schedule_cron | 10 1 * * * | Australia/Sydney 每天 01:10 |
| timezone | Australia/Sydney | 本机运行时区 |

## 设备参数

| 参数 | 当前值 | 说明 |
|---|---|---|
| device_key | airM2 | 设备唯一键 |
| expected_model_keywords | MacBook Air | 以本机事实为主 |
| expected_chip_keywords | M2 | 以本机 Apple M2 为准 |
| expected_memory_gb_min | 8 | 本机 8GB |
| retention_days | 3 | 本机仅保留最近 3 天 |

## 风险阈值

| 参数 | 当前值 | 说明 |
|---|---|---|
| storage_yellow_free_gb | 70 | 剩余空间低于该值黄色预警 |
| storage_red_free_gb | 40 | 剩余空间低于该值红色预警 |
| swap_yellow_gb | 1 | swap 达到该值黄色预警 |
| swap_red_gb | 4 | swap 达到该值红色预警 |

## 清理边界

| 参数 | 当前值 | 说明 |
|---|---|---|
| macdata_retention_cleanup | true | 远程验证成功后清理 3 天以前 macdata 自身数据、报告、运行记录 |
| archive_push_cache_cleanup | true | 远程验证成功后清理 `data/cache/archive_push` 临时浅克隆缓存 |
| docker_cleanup | false | 原始任务包明确不自动清理 Docker |
| homebrew_cleanup | false | 原始任务包明确不自动清理 Homebrew |
| user_cache_cleanup | false | 原始任务包明确不自动清理系统/用户缓存 |
| project_cache_cleanup | false | 原始任务包明确不自动清理项目缓存 |

## Automation C 事务参数

| 参数 | 当前值 | 原理与门禁 |
|---|---|---|
| base_branch | main | 唯一持久分支 |
| legacy_archive_branch | macdata-airM2 | 仅用于检测并拒绝旧生产分支，不作为写入目标 |
| transaction_branch_prefix | automation-c/macdata-airM2 | 每个 run/stage 生成唯一、确定性的短命分支名 |
| transaction_task_id | TSK.OpenAIDatabase.CLEAN1.0001 | PR marker 与治理路由键 |
| transaction_acceptance_id | ACC.OpenAIDatabase.CLEAN1.0001 | Settlement 验收键 |
| settlement_timeout_seconds | 900 | 等待 trusted Settlement 的最长时间，超时 fail closed |
| settlement_poll_seconds | 10 | Settlement 状态轮询间隔 |
| required_at_rest_state | PR/Issue/non-main = 0/0/0 | 运行前后必须满足，避免事务叠加和残留 |
| issue_mutations | 0 | 生产者不创建、关闭或修改 Issue |
| verification | exact per-file SHA256 manifest | `main` 必须与本次源快照逐文件一致才允许后续清理 |
