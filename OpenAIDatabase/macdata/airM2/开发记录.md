风险与管控：

1. GitHub 分支不存在：由首次 push 自动创建，随后用远程 hash 验证。
2. 清理误删风险：仅允许配置白名单和可重建缓存；不使用 sudo；不删除 volumes、源码、数据。
3. 凭证泄露风险：上传前扫描 raw/report/status；命令输出会按高风险模式脱敏。
4. 设备不匹配风险：预检失败时停止。

验收标准：

1. 单元测试通过。
2. 预检通过。
3. `main` 上任务包 commit + push 成功。
4. `macdata-airM2` 归档分支创建或更新成功。
5. 远程 hash 与本地提交 hash 一致后，才允许清理旧本机数据和受控缓存。
