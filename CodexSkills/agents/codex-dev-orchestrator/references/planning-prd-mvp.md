# Planning, PRD, and MVP Scope

Use this reference only for `PLAN_SCOPE` requests.

## Minimum Output

Produce a concise planning artifact with:

1. Goal and user outcome.
2. Target users and main workflow.
3. In scope.
4. Out of scope.
5. Core pages, modules, or services.
6. Data inputs and storage.
7. Risks and constraints.
8. Acceptance criteria.
9. Stop conditions.
10. Recommended first run contract.

## Interaction Style

- Prefer numbered choices, defaults, and compact matrices.
- Ask only for missing decisions that materially change implementation.
- When the repo or task pack already answers a question, record it as an
  assumption instead of asking again.

## PRD Shape

Use this structure unless the user provides another one:

```text
01 Purpose
02 Users and workflows
03 Scope and non-scope
04 Functional requirements
05 Data and persistence
06 UI/UX requirements
07 Validation and acceptance
08 Stage plan
09 Open questions
```
