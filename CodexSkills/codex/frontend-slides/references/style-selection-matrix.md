# 风格选择矩阵

用于自动选择 HTML slides / 网页 PPT 的视觉方向。用户未明确要求先选风格时，不要提问阻塞执行；按下表选择默认方案并在最终回复说明。

| 场景 | 适用场景 | 推荐模板/风格 | 默认版式倾向 | 避免事项 |
|---|---|---|---|---|
| 商务汇报 | 内部经营复盘、咨询汇报、管理层同步 | `blue-professional`、`cartesian`、瑞士国际主义 | 浅色、清晰标题、KPI 卡片、2-3 列信息结构 | 避免过度装饰、霓虹、低对比文字 |
| 产品发布 | 新产品、功能发布、demo day、SaaS 发布 | `signal`、`studio`、`neo-grid-bold`、guizang 瑞士风 | 强封面、功能分段、产品截图大图、节奏感强 | 避免密集表格和弱主视觉 |
| 数据看板 | 指标复盘、运营数据、财务/业务看板 | `long-table`、`cartesian`、`cobalt-grid`、瑞士 Data Hero | 高密度网格、图表优先、固定 16:9、少动效 | 避免杂志式大留白导致信息不足 |
| 投资/融资材料 | 融资路演、投资备忘、投资人更新 | `blue-professional`、`monochrome`、`broadside` | 高正式度、问题-方案-市场-进展-财务路径 | 避免玩具感、过度复古、夸张营销语 |
| 技术方案 | 架构方案、工程路线、系统设计、API/平台说明 | `cobalt-grid`、`cartesian`、`signal`、guizang 瑞士风 | 深/浅均可，图解、流程、对比、代码片段区块 | 避免只做抽象概念图，不给结构细节 |
| 品牌展示 | 品牌手册、设计提案、案例集、视觉系统展示 | `studio`、`editorial-tri-tone`、`bold-poster`、guizang 电子杂志风 | 大标题、图像网格、品牌色块、案例页 | 避免默认商务蓝和无差异卡片 |
| 极简高级感 | 高端服务、战略叙事、董事会/合伙人材料 | `monochrome`、`soft-editorial`、`biennale-yellow` | 低密度、大留白、精确字号层级、克制动效 | 避免多彩渐变、复杂纹理、信息堆叠 |
| 强视觉营销感 | Campaign、增长活动、品牌传播、创意 pitch | `bold-poster`、`block-frame`、`creative-mode`、`coral` | 大字报、强对比色、海报感、少量关键句 | 避免长段正文和弱 CTA |
| 深色主题 | 技术、开发者、web3、夜场发布、沉浸式演讲 | `signal`、`cobalt-grid`、`8-bit-orbit`、`broadside` | 暗底高亮、发光线条、代码/图表清晰对比 | 避免低对比灰字和纯装饰暗背景 |
| 浅色主题 | 正式汇报、打印/转 PDF、管理层阅读 | `blue-professional`、`cartesian`、`soft-editorial` | 白/暖纸底、深色标题、表格清晰 | 避免浅色上低饱和细字 |
| 信息密度高 | 异步阅读、报告型 slides、表格/指标多 | `long-table`、`cartesian`、`block-frame` | 多栏网格、表格、注释、分节页更多 | 避免把所有内容塞进单页小字 |
| 信息密度低 | 现场演讲、keynote、愿景叙事 | `bold-poster`、`biennale-yellow`、`editorial-forest`、guizang 电子杂志风 | 一页一观点、强封面、金句页、视觉节拍 | 避免连续项目符号和小字号说明 |

## 默认决策顺序

1. 先按用途：商务/技术/数据/投资优先可读性；产品/品牌/营销优先视觉记忆点。
2. 再按受众：高层和投资人偏正式；开发者和创意团队可更强视觉。
3. 再按载体：需要 PDF/打印默认浅色；现场大屏可选深色。
4. 再按密度：资料越多越靠近 `long-table`、`cartesian`；口播越多越靠近 `bold-poster`、`biennale-yellow`。
5. 没有任何偏好时，默认 `blue-professional` 或 guizang 瑞士国际主义。
