# Awesome Design Systems

Use the bundled `references/source/README.md` as a local catalog of public design systems and pattern libraries.

## Workflow

1. Search the catalog for the user's target company, platform, industry, or UI need:
   - `rg -i "<query>" references/source/README.md`
   - Useful query terms: `Components`, `Voice & Tone`, `Designers Kit`, `Source code`, company names, framework names, and industry names.
2. Read only the relevant rows and nearby context from `references/source/README.md`.
3. Use the catalog to shortlist design systems, then inspect official linked sources when current accuracy matters.
4. For implementation tasks, translate selected system principles into concrete UI choices: layout density, component hierarchy, typography, color use, interaction states, and documentation/source-code references.

## Constraints

- Do not load the whole catalog unless the user explicitly asks for a full inventory.
- This is a snapshot of `alexpate/awesome-design-systems`; verify official links for current docs, source repositories, or licensing.
- Do not treat a listed design system as permission to copy proprietary visual assets.
