# Changelog

## v0.0.0.4 — 2026-07-23

- 目标人物 Skill 改为直接调用：用户无需选择身份、编号或权重。
- 运行路由器根据当前任务自动选择或组合已蒸馏身份分面，并保持最小文件加载。
- 删除逐次调用的 `0.0.0.N` 版本、身份门、编号运行目录和版本化文件名。
- 以可选无编号审计记录替换旧调用计数器，默认只保存任务 hash 与结果元数据。
- 将 `0.0.0.1` 至 `0.0.0.999` 定义为人物产物版本；每个 canonical 人物独立、连续计数。
- 产物版本只在成功登记时正式占号；失败不占号，并发登记受跨进程锁保护。
- 保留显式 `$skill` 调用策略、七类唯一登记、安全边界与 `~/.codex/skills` 单路径安装。

## v0.0.0.3 — 2026-07-23

- 仓库定制分发将中文展示名改为“人物蒸馏 Skill”，版本号保持不变。
- Codex 默认安装路径恢复为 `~/.codex/skills`，并拒绝与 `~/.agents/skills` 同名双安装。
- 增加七类人物产物唯一登记、完整发布 ZIP 保存、跨目录去重和机器校验。
- 输入合同改为“姓名 + 身份”，场景可选并自动路由。
- 身份合并为六个主身份和一个多重身份；支持中文、编号、别名和权重表达。
- 增加每次调用身份门、原子 `0.0.0.N` 运行版本、失败留痕和并发安全。
- 从“人物画像”升级为能力、认知、决策、策略、Work、Persona、负能力、分歧地图和执行循环。
- 分离稳定人物模型、用户适配、程序性经验和 episodic 运行记忆。
- 增加任务完成、工具使用、能力校准、匿名人物、长程稳定和 token 效率评测。
- 原始发布曾将 Codex/Agent Skills 默认目录统一为 `~/.agents/skills`；本定制分发已恢复 Codex 专属目录，仍保留 `agents/openai.yaml` 和显式调用策略。
- 完成十轮改进及两轮六角色隔离复审；新增供应链、秘密、包结构和全新环境验证门。
