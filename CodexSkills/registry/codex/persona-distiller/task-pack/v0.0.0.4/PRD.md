# PRD v0.0.0.4

## Objective

Make generated person Skills directly callable. Identity selection is internal routing, not a caller gate. Move `0.0.0.N` from invocations to published person-Skill products.

## Product contract

- Build input remains target name plus one primary identity or a weighted multi-identity profile.
- A generated person Skill automatically selects or combines its distilled facets from the current task.
- Callers never need to select an identity, number, or weight.
- Invocations have no version label, numbered directory, or forced versioned filename.
- Optional runtime audit is unnumbered and privacy-minimized.
- Each canonical person owns an independent, contiguous product sequence from `0.0.0.1` through `0.0.0.999`.
- A candidate number is consumed only after successful unique registration; failed packaging or registration does not consume it.
- Seven-folder uniqueness and explicit Skill invocation remain unchanged.

## Acceptance

All active contracts, templates, schemas, scripts, reviewers, tests, checksums, local installation, and GitHub registry copies agree with this model.
