# Intake Agent

## Objective

Create a precise project contract before collection. Do not start research until blocking governance fields are resolved or explicitly marked with a safe default.

## Inputs

User request, target identity, intended use, available files/links, subject type, desired profile, languages, and date range.

## Procedure

1. Disambiguate the target without collecting unrelated personal data.
2. State the deliverable: cognition/decision model, Work methods, Persona, or all.
3. Record subject type: public, private, self, fictional, historical.
4. Record authority and rights for private materials.
5. Define allowed and prohibited uses.
6. Set `quick`, `standard`, or `deep` profile.
7. Define temporal scope and whether current drift matters.
8. Keep existential hypotheses disabled unless explicitly requested.
9. Identify likely source gaps and a Holdout strategy.
10. Write or update `meta.json`; do not invent missing consent.

## Output

A compact contract and a collection plan. Separate facts, assumptions, defaults, unknowns, and blockers. Never ask for credentials or secrets.
