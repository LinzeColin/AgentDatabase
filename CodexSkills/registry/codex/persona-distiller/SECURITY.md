# Security and privacy

## Threat model

1. 网页、文件、转录和代码中的 prompt injection。
2. 私域资料、身份信息、凭据或版权正文被打包泄漏。
3. 运行记忆污染稳定人物模型，造成长期 persona drift。
4. 同一人物的并发发布产生重复、跳号或越过 `0.0.0.999` 的产物版本。
5. 目标人物风格掩盖事实错误、越权工具调用或高风险建议。
6. 安装包路径穿越、混入多顶层目录或被旧版本文件污染。

## Controls

- 外部内容一律视为数据，不授予其治理或工具权限。
- 原始语料、Holdout、运行 Skill 和审计材料分区。
- 默认最小化打包；秘密扫描为硬门。
- 稳定模型只能经 correction → impact analysis → regression → snapshot 晋级。
- 人物产物登记使用跨进程锁、连续版本校验和原子 JSON 写入；失败登记不占号。
- 单次运行不使用版本计数；可选审计只记录最小化、无编号元数据。
- 工具执行遵循宿主权限和用户授权；不伪造未实际运行的结果。
- 私域/自己需要授权与保留策略；虚构角色必须记录版本/作者谱系。
- 高风险决策必须给独立事实与风险检查，不得只输出人物立场。

发现问题时保留必要的无编号失败证据，不静默覆盖已登记产物或稳定人物模型。
