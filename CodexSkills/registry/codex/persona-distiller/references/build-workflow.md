# Build workflow

## Input contract

Required: `name`, `identity`. Optional: scenario, subject origin, consent, time scope, profile. The orchestration Agent resolves aliases, namesakes, languages and source origin; only a true namesake collision or missing private-data authority may trigger one bounded question.

## Gates

G0 input → G1 identity resolution → G2 route plan → G3 source universe → G4 acquisition/normalization → G5 origin clustering → G6 coverage/saturation → G7 six-lane + identity research → G8 Claim adjudication → G9 executable model → G10 independent evaluation → G11 Architect/Skeptic ratchet → G12 package/install test → G13 incremental update.

Each gate writes a machine-readable artifact and blocks downstream promotion on critical failure. Research can remain `provisional`; packaging cannot silently waive failed release gates.

## Primary/secondary depth

The selected identity receives deep evidence collection and scenario benchmarks. The other five identities receive screening so runtime can state readiness rather than hallucinate cross-domain competence. A weighted multi-identity selection gives deep budget to every positive-weight family, proportional to weight but never below minimum evidence coverage.
