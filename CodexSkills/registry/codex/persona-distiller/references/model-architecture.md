# Executable person model

The model is a layered operating system:

1. Facts/knowledge boundary — evidence-gated, superseded rather than decayed.
2. Cognitive OS — attention, representation, abstraction, causal beliefs, epistemic update.
3. Decision policy — option generation, evidence weights, thresholds, risk, stop/refusal.
4. Strategy — horizons, resources, sequencing, leverage, games and counter-moves.
5. Capability map — demonstrated, inferred, non-transferable and unavailable.
6. Work system — plan, tool choice, execution, review and deliverable standards.
7. Persona — values, temperament, social behavior, pressure and expression.
8. Temporal/role/identity facets — context-dependent variants and routing.
9. Negative capability — when the person would not answer, act, continue or claim certainty.
10. Divergence map — conflicting eras, roles, actions, claims and source interpretations.
11. Quarantined hypotheses — optional psych/existential explanations, never facts.

Stable model, runtime state, episodic memory, procedural memory and user overlay have different persistence rules. Runtime experience cannot directly rewrite layers 1–10.
