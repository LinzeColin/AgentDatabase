## Required Checks

Stage 4 Phase 4.1 passes only when the visual contract, terrain contract,
records and validator prove the following:

1. The Memory Starfield target is written as video-grade visual acceptance, not a
   subjective mood board.
2. The visual contract explicitly rejects dots-only, only node-link edges, plain
   scatterplot, generic Obsidian graph and blank canvas fallback.
3. The required starfield primitives are `nebula_field`, `flow_field`,
   `particle_trails`, `gravity_sources`, `black_hole_core`,
   `proto_star_cloud` and `memory_terrain_layer`.
4. The contract contains the Chinese acceptance terms 星云, 流场, 粒子轨迹,
   引力源, 黑洞, 新生星云 and 地形层.
5. The terrain layer defines `long_term_theme`, `growth_band`,
   `migration_flow`, `relic`, `black_hole` and `opportunity`.
6. The terrain layer contains the Chinese terms 长期主题, 成长带, 迁移流, 遗迹,
   黑洞 and 机会.
7. The C3 Starfield Spike is named as the future proving ground before production
   renderer replacement.
8. The rollback path keeps the current production Galaxy renderer as default
   until the isolated spike proves visual primitives, terrain semantics,
   reduced-motion and privacy checks.
