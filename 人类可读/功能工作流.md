# Memory Atlas 功能工作流

## 结论

Memory Atlas 的日常流程是“同步真实来源 → 生成派生分析 → 浏览并核对证据 → 形成提案 → 人工授权后验证应用”。任何一步缺少真实输入、证据或授权时都应停止，不生成伪数据，也不把候选结论当成事实。

所有命令默认从 `OpenAIDatabase` 目录运行。先用 `--dry-run` 查看计划，确认输入、写入范围和停止条件后再决定是否执行。

## 操作

### 1. 查看每日运行计划

```bash
python3 scripts/atlasctl.py run --profile owner-daily --dry-run
```

如只需要检查某一步，可增加 `--step`，可选值包括 `sync`、`analyze`、`build-atlas`、`audit`、`push`、`proposals`、`generate-personalization-prompt` 和 `deep-explore`。

### 2. 同步一个真实来源

```bash
python3 scripts/atlasctl.py sync --source chatgpt --dry-run
python3 scripts/atlasctl.py sync --source codex --dry-run
python3 scripts/atlasctl.py sync --source future-agent --dry-run
```

ChatGPT 使用官方导出作为可靠回退；Codex 使用本地 session 来源；其他 Agent 必须先登记来源并提供本地输入。没有真实输入时命令只能返回合同，不能制造记录。

### 3. 生成派生分析和星图

```bash
python3 scripts/atlasctl.py analyze --stage facets --dry-run
python3 scripts/atlasctl.py analyze --stage clusters --dry-run
python3 scripts/atlasctl.py build-atlas --dry-run
```

`facets` 把记录整理成来源、主题、意图、任务、项目、产物、语言、工具、摩擦和价值信号；`clusters` 把事件聚合为主题簇和层级簇。派生结果必须保留证据引用或明确的证据缺失原因。

### 4. 识别返工、机会和投入价值

```bash
python3 scripts/atlasctl.py analyze --stage low-value-loops --dry-run
python3 scripts/atlasctl.py analyze --stage opportunities --dry-run
python3 scripts/atlasctl.py analyze --stage information-roi --dry-run
python3 scripts/atlasctl.py analyze --stage formula-what-if --dry-run
```

低价值循环只描述可观察到的重复返工、讨论未落地、过度优化和范围扩张，不做心理诊断。机会输出必须说明“为什么现在不做”和一个最小验证动作。公式预览只比较权重影响，不直接写入有效配置。

### 5. 生成个性化提示和深度探索

```bash
python3 scripts/atlasctl.py generate-personalization-prompt --dry-run
python3 scripts/atlasctl.py chatgpt-deep-explore --mode prefill_only --dry-run
```

个性化输出面向 ChatGPT、Codex 和其他 Agent，来源必须是低敏派生报告。深度探索默认 `prefill_only`，需要用户主动触发；`--open` 只打开预填充页面，不代表自动发送。

### 6. 复核提案

```bash
python3 scripts/atlasctl.py proposals --help
python3 scripts/atlasctl.py apply --help
```

提案主路径是 `draft → pending_human_review → approved_by_human → applying → applied → validated → committed`。验证失败进入 `rollback_or_needs_revision`。真实提案必须由人类检查目标文件、风险、差异、验证命令和回滚方案后才能授权。

## 判断标准

- 有真实输入、有来源登记、有证据引用，才能把派生结论提升为可复核结果。
- 有明确人类问题和下一步行动，图表才值得进入核心视图。
- 有人工授权、目标范围允许、验证通过和回滚能力，提案才可以应用。
- raw、凭证、浏览器登录态和远端 push 不属于默认应用范围。
- 当前任务边界禁止写入时，即使命令具备 `--apply` 也不能执行。
