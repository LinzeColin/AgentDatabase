# Memory Atlas 维护与故障处理

## 结论

日常维护只需要四个验证入口和少量故障定位命令。先运行低成本检查，再按同步、界面或发布风险扩大范围；不要恢复旧的逐 Stage validator，不要在故障时修改 raw、关闭安全门禁或直接 push。

本轮 v1.2.1 尚未最终交付，任何本地通过都不代表线上网站或已安装应用已经更新。

## 操作

### 验证顺序

在 `OpenAIDatabase` 目录运行：

```bash
npm run validate:fast --prefix apps/memory-atlas
```

涉及数据来源、manifest 或 raw 边界时：

```bash
npm run validate:sync --prefix apps/memory-atlas
```

涉及布局、浏览器交互或用户路径时：

```bash
npm run validate:ui --prefix apps/memory-atlas
```

准备发布候选时：

```bash
npm run validate:release --prefix apps/memory-atlas
```

相关代码变更还应运行：

```bash
npm run build --prefix apps/memory-atlas
PYTHONPATH=.. python3 -B -m unittest discover -s tests -q
```

`validate:release` 的计划通过不等于真实部署通过；最终仍需准确 commit 的安装、部署、在线多视口和恢复验收。

### 页面读取失败

1. 确认开发服务仍在运行，终端没有构建错误。
2. 确认 `apps/memory-atlas/public/memory_atlas.json` 或发布候选中的快照存在。
3. 运行 `npm run build --prefix apps/memory-atlas`，不要用旧 bundle 文件名判断版本。
4. 比较 source commit、runtime 的 `memory_atlas_build.json` 和当前 `git rev-parse HEAD`。
5. 仍失败时保留浏览器 console、network 和启动器日志证据。

启动器日志位于：

```text
~/Library/Logs/OpenAIDatabase/memory-atlas-launcher.log
```

### 筛选后没有结果

先重置全部筛选，再依次恢复分析对象、时间、主题和搜索词。若单一条件导致空结果，检查该条件是否仍存在于当前快照，不要生成伪记录填充空状态。

### WebGL 不可用

确认浏览器允许硬件加速。无法恢复时使用静态降级说明、关系网络、搜索和证据入口继续工作；不要把空白 Three.js 画布当成验证通过。

### 提案不能应用

依次检查：是否为 `approved_by_human`、是否过期、目标是否在允许范围、是否触及 raw 或凭证、验证命令是否存在、回滚计划是否完整。任一条件失败都保持只读，不手工改状态绕过门禁。

### 同步失败

1. 运行对应来源的 `sync --dry-run`。
2. 检查真实输入是否存在、来源是否登记、官方导出是否可解析。
3. 运行 `validate:sync` 和隐私扫描。
4. 检查 manifest 缺项、hash drift、已登记文件删除和凭证命中。
5. 修复输入或合同后重跑；不要覆盖旧 raw，也不要关闭 credential gate。

### 旧命令失效

公共验证只保留 `validate:fast`、`validate:sync`、`validate:ui` 和 `validate:release`。历史 alias 只能通过只读迁移表查询替代命令，兼容期到 v1.2.2 结束；不要把旧 alias 重新加入 `package.json`。

## 回滚

- 文档、代码或配置任务失败：只 `git revert` 对应本地 task commit。
- 提案应用失败：使用提案自带 rollback plan 或对应 Git commit；raw 不参与回滚。
- runtime 与 source 不一致：停止旧服务，用准确 source commit 重新构建或重装，不复制未知缓存。
- 同步或隐私门禁失败：保留日志和输入 hash，停止归档与备份，不删除原始来源。
- 最终发布失败：保持线上旧版本，回滚发布候选；不能把旧站可访问当成新版本成功。

回滚后重新运行触发失败的最小检查，再按 `fast → sync/ui → release` 扩大验证。当前 Task Pack 全部完成前，不清理 TaskPack、Roadmap、rescue bundle、运行证据或机器缓存。
