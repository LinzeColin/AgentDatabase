# Memory Atlas 快速开始

## 结论

Memory Atlas 是本地优先的记忆决策台。它把 ChatGPT、Codex 和其他已登记来源的资料整理成低敏派生快照，帮助你先看变化，再核对证据，最后决定下一步。默认页面不会直接修改原始资料，也不会替你批准提案或推送 GitHub。

v1.2.1 仍在本地开发阶段。本轮只收敛说明文件；线上网站、已安装的 `Memory Atlas.app` 和 GitHub `main` 都没有因此更新。只有完整 Task Pack、复审和最终交付全部通过后，才允许统一推送、重装和部署。

## 操作

### 1. 打开本地开发版

在 `OpenAIDatabase` 目录运行：

```bash
npm run dev --prefix apps/memory-atlas
```

终端会显示本地地址。打开该地址后，先确认页面顶部能看到“记忆决策台”，并且快照状态不是读取失败。

如果本机已经安装正式启动器，也可以直接打开 `Memory Atlas.app`。启动器使用 `http://127.0.0.1:4177`，但它显示的是最近一次已安装版本，不等于当前工作树。

### 2. 用三分钟完成一次判断

1. 在“发生了什么”查看新增资料、增强或减弱的结论、待授权提案和同步失败。
2. 选择“哪些主题在变化”或“变化何时发生”，找到对应主题和时间证据。
3. 点击记录后查看证据入口，确认含义、影响、来源和未来用途。
4. 在“查找与核对”缩小范围，在“决定下一步”形成一个明确动作。
5. 结论不准确时只生成提案；先复核差异、验证和回滚，再决定是否授权。

### 3. 先做低成本检查

```bash
npm run validate:fast --prefix apps/memory-atlas
```

只有需要验证同步、浏览器界面或发布候选时，才分别运行 `validate:sync`、`validate:ui` 或 `validate:release`。旧的逐 Stage `validate:*` 命令已经退出公共入口，不应继续照抄历史文档。

## 你应该看到什么

- 左侧导航按“判断、探索、复盘”组织，不要求先理解内部数据结构。
- 顶部筛选可按分析对象、层级、分类、主题和搜索词缩小范围。
- 默认信息先解释“发生了什么、为什么重要、下一步做什么”。
- `schema`、ID、hash、机器字段和证据路径默认放在高级详情，不占据主阅读路径。
- 页面为空、筛选无结果、WebGL 不可用或提案不可写时，应直接给出原因和可执行动作。

## 安全边界

- 原始归档只追加，不覆盖、不删除；提案不能把 raw 目录作为写入目标。
- cookie、session token、password、API key、private key、OAuth token 和浏览器凭证库不是记忆，不能进入 GitHub。
- ChatGPT 深度探索默认只预填，不静默发送。
- 未经人类授权的提案不能应用；应用后验证失败必须回滚或进入待修订状态。
- 当前 v1.2.1 任务全部完成前，不推送、不部署、不重装应用，也不清理恢复资料。
