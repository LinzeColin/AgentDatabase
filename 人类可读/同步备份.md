# Memory Atlas 同步与备份

## 结论

同步的目标是把真实来源变成可恢复、可审计的数据链，而不是抓取账户状态。用户授权后的 transcript 可以进入公开 raw 归档；凭证和账户控制信息永远不能进入 GitHub。raw 采用只追加合同，已登记文件不得覆盖、修改或删除。

当前 v1.2.1 Task Pack 尚未完成，因此本地可以检查同步和备份计划，但不得提前远端 push、部署或清理恢复资料。

## 操作

### ChatGPT

先检查合同：

```bash
python3 scripts/atlasctl.py sync --source chatgpt --dry-run
```

有官方导出文件时，先做只读解析：

```bash
python3 scripts/atlasctl.py sync --source chatgpt --official-export /path/to/export.zip --dry-run
```

浏览器能力只允许读取已登录状态下的会话标题、内容和必要 metadata；遇到密码、验证码、cookie 或 session token 必须停止。浏览器路径不可用时，使用官方导出，不绕过登录或安全门禁。

### Codex

```bash
python3 scripts/atlasctl.py sync --source codex --dry-run
```

Codex 同步读取本地 session 来源，生成受策略约束的 public raw、低敏派生摘要和同步日志。不要把 `.env`、tokens、API keys、本机缓存或应用私有状态当成对话资料。

### 其他 Agent

```bash
python3 scripts/atlasctl.py sync --source future-agent --agent-id example --input /path/to/input.json --dry-run
```

新来源必须先登记 `source_id`、来源类型、输入方式、raw 根目录和凭证边界。没有本地输入时不得生成伪记录；不同 Agent 的 raw 必须放在各自目录，不能混写。

## 归档规则

| 内容 | 允许位置 | 规则 |
|---|---|---|
| ChatGPT transcript | `data/public_raw/chatgpt/` | 用户授权后公开，只追加。 |
| Codex transcript 或受控摘要 | `data/public_raw/codex/` | 按当前同步策略生成，只追加。 |
| 其他 Agent 资料 | `data/public_raw/agents/{agent_id}/` | 必须先登记来源，只追加。 |
| 派生分析 | `data/derived/` | 可以重建，不可冒充原始资料。 |
| 运行日志 | `data/run_logs/` | 记录输入、结果、失败和证据，不写凭证值。 |
| 凭证与登录态 | 不允许进入 tracked 路径 | 发现即失败关闭并停止备份。 |

机器账本记录 source、文件、hash 和导入时间，用于发现 hash drift、文件删除或未登记新增。README、`.gitkeep` 和系统占位文件不应被当成 transcript 锁入账本。

## 本地备份检查

```bash
python3 scripts/atlasctl.py push --dry-run
```

该命令只显示备份范围和 Git 状态，不 stage、不 commit、不 push。`--apply` 只允许在当前 run contract 明确授权时创建本地提交；它仍不等于远端 push。v1.2.1 全部任务和复审完成前，不使用它绕过最终统一交付门禁。

备份候选通常覆盖 `data/public_raw`、`data/derived`、`data/run_logs`、`docs/reviews` 和 `reports`。提交前必须确认没有凭证、未授权私密内容、本机绝对路径和与当前任务无关的用户变更。

## 恢复检查

1. 从 GitHub 的准确 commit 恢复代码和允许公开的数据。
2. 校验 manifest 与 SHA-256，不以文件名或旧聊天记录代替完整性证明。
3. 安装依赖并重新生成派生快照。
4. 运行 `validate:fast`、必要的同步验证、前端 build 和浏览器关键路径。
5. 比较恢复后的 source commit、runtime commit 和发布 commit；不一致时不得宣称恢复完成。

原始 TaskPack、Roadmap、rescue bundle 和不可变发布证据在整个项目完成前保留，不作为日常缓存删除。
