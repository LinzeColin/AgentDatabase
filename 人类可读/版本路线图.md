## 交付门禁

- 每个 Task 都要有最小实现、验证结果、风险、回滚和本地 commit。
- 每个 Stage 完成后复审；问题修复通过后才能进入下一 Stage。
- 全部 Task 完成后再做整体复审和恢复证明。
- 只有最终验收通过，才统一 push GitHub `main`、重装本地应用和部署线上版本。
- 中间不创建 branch 或 PR，不留下远端过程状态。
- 原始 TaskPack、Roadmap、恢复 bundle、不可变 manifest 和必要运行资料保留到整体任务完成。

## 停止条件

遇到任务范围不清、真实输入缺失、凭证风险、raw 不可变性失败、工作树混入未知变更、验证失败或恢复证据不完整时，停止当前写入并保留事实。不能用文档存在、字符串命中或旧 PASS 记录替代真实产品验收。
