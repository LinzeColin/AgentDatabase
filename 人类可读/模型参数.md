# Memory Atlas 模型与参数

## 结论

Memory Atlas 的行为智能和经济指标都是内部 proxy，用于排序、比较和提出下一步，不是收入预测、财务建议、心理诊断或对个人能力的绝对评价。模型结果必须能回到来源、参数和证据；任何权重变化先生成提案或预览，不直接修改有效配置。

公式和参数的可执行真相位于 `机器治理/参数与公式/`。本文件解释怎么理解和使用，不复制全部机器配置。

## 操作

### 1. 先看六个经济信号

| 信号 | 说明 | 使用方式 |
|---|---|---|
| 时间节省 | 估算自动化、模板化、产品化和减少返工可能节省的重复工作。 | 用于排序候选，不作为工时结算。 |
| 复用价值 | 判断主题、脚本、模板、validator 或产品入口是否值得沉淀。 | 优先核对跨时间重复和证据密度。 |
| 返工成本 | 汇总重复返工、讨论未落地、过度优化和范围扩张。 | 高分时先收紧完成标准和停止条件。 |
| 机会分 | 结合候选机会分数和证据数量判断可行动程度。 | 证据不足时先做更小验证。 |
| 技能复利 | 判断跨时间重复的能力和工作流是否值得长期固化。 | 与真实交付和复用结果一起复核。 |
| 自动化与增强比例 | 区分适合自动执行和适合保留人工判断的工作。 | 不把所有机会都写成自动化任务。 |

生成或检查经济 proxy：

```bash
python3 scripts/atlasctl.py analyze --stage economic-proxy --dry-run
```

如当前 CLI 不支持该 stage，以 `python3 scripts/atlasctl.py analyze --help` 的实际列表为准，不凭历史文档发明命令。

### 2. 用信息 ROI 判断内容是否进入主视图

信息 ROI 由决策价值、可行动性、证据强度、新鲜度和复用价值共同决定，再除以阅读、导航、误导和维护成本。核心规则是：没有明确人类问题、行动用途和证据来源的图表不能进入主决策视图。

```bash
python3 scripts/atlasctl.py analyze --stage information-roi --dry-run
python3 scripts/atlasctl.py audit --check visual-roi
```

证据等级从 A 到 X：A 表示多来源多时间点一致；B 表示单来源多次出现；C 表示新兴信号；D 表示假设待验证；X 表示冲突或反证强。等级低不等于删除，应降低结论强度并补证据。

### 3. 只预览参数变化

```bash
python3 scripts/atlasctl.py analyze --stage formula-what-if --dry-run
python3 scripts/atlasctl.py audit --check formula-what-if
```

可调整的权重包括时间节省、复用价值、机会价值、技能复利、自动化匹配、返工成本和低价值循环惩罚。当前有效边界是：权重范围 `0.25` 到 `2.0`，步长 `0.05`；预览分数限制在 `0` 到 `100`；`active_config_write=false`；应用前必须生成提案并经人类授权。

### 4. 复核协作与授权指标

协作质量关注规划清晰度、执行清晰度、复审负担、返工控制、范围清晰度、可测试性和可回滚性。它用于改进工作流，不用于评价人格。

人类负责目标、范围、业务优先级、高风险取舍和是否授权；Agent 负责在边界内执行、保留证据、运行验证并说明回滚。raw、凭证和未授权目标不能进入应用范围。

## 参数变更规则

1. 先说明要改善的用户判断，不从“想调参数”开始。
2. 找到唯一有效配置、公式 ID 和当前默认值。
3. 用真实派生数据运行 what-if，记录基线与变化。
4. 检查收益是否只是把噪声放大，是否增加误导或维护成本。
5. 生成包含目标文件、差异、验证和回滚的提案。
6. 人类授权后再应用；验证失败立即回滚或进入待修订。

代码字段、公式 ID、参数 key 和 API 名称保持英文，中文只负责解释其含义和操作。
