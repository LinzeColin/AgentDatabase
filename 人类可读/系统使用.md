# Memory Atlas 系统使用

## 结论

使用 Memory Atlas 时不需要逐页浏览全部图表。先选择要回答的问题，再进入对应视图；所有视图共享当前筛选和选中记录，证据应能在星图、时间轴、搜索、复盘和详情之间连续核对。

默认界面使用中文解释。代码标识、API、数据字段和专业模型名保留英文，是为了与实现一致；它们不应替代人类结论。

## 操作

### 判断：我现在应该先判断什么

| 入口 | 回答的问题 | 建议动作 |
|---|---|---|
| 发生了什么 | 上次查看后有哪些新增、增强、减弱、待授权或同步失败？ | 先选一条最重要变化，再去找证据。 |
| 决定下一步 | 哪个结论已经足够支持行动，哪个还需要补证据？ | 只保留一个明确下一步、预期产物和停止条件。 |

首页还会汇总记忆天气、风险黑洞、新生机会、层级资产、主题结构和问题地图。它们用于排序注意力，不是对个人状态的诊断。

### 探索：我需要从哪里找证据

| 入口 | 适用场景 | 建议动作 |
|---|---|---|
| 哪些主题在变化 | 看主题簇、关系和近期活跃重心。 | 点击星体，核对代表记录和相邻主题。 |
| 资料如何关联 | 看来源、画像、项目决策和行动之间的结构。 | 从一个节点向上找背景，向下找行动。 |
| 变化何时发生 | 看结论增强、减弱或冲突出现的时间。 | 选定时间窗口，再回到星图或搜索。 |
| 查找与核对 | 已知关键词、项目或规则，需要快速定位。 | 缩小搜索词和筛选，打开结果证据。 |

### 复盘：哪里值得投入或降噪

| 入口 | 适用场景 | 建议动作 |
|---|---|---|
| 哪里值得投入 | 比较时间节省、复用、返工、机会和技能复利信号。 | 优先看高行动价值且证据足的项目。 |
| 关系网络 | 关系结构比单条记录更重要时。 | 识别中心节点、孤立证据和冲突连接。 |
| 投入节奏 | 判断某段时间是否集中、间断或过载。 | 对照实际交付，不把活跃度等同于价值。 |
| 反复出现什么 | 快速发现高频主题。 | 把词频当线索，再用搜索核对原始语境。 |

## 筛选方法

1. 先选分析对象：总数据源、ChatGPT 或 Codex。
2. 再选时间范围；没有明确问题时不要同时叠加多个筛选。
3. 需要聚焦项目时使用主题或搜索词，确认结果数量没有被筛空。
4. 点击任意记录后，检查其他视图是否保持同一选中对象和时间上下文。
5. 没有结果时先重置筛选，再逐项恢复条件，定位是哪一项过窄。

## 如何读图

- 多维图谱用于定位主题重心、高频分布和值得继续追问的簇。
- 经济图谱用于比较任务投入、自动化与增强、信息 ROI 和机会证据，不做收入预测。
- 治理图谱用于发现返工、范围扩张、证据缺口、授权风险和公式影响。
- 每张进入主路径的图都应说明“回答什么问题”和“看完做什么”；没有决策价值的装饰图不进入核心视图。

## 证据与高级详情

主页面先显示中文结论、重要性和下一步。需要复核时再展开高级详情，查看 `evidence_refs`、来源、时间、提案状态和机器字段。机器字段只能支持结论，不能替代结论。

如果结论与证据不一致，停止后续应用动作，回到搜索和时间轴核对来源；仍有冲突时生成待复核提案，不直接改长期记忆。
